Fruits Website

Details

Project Name: Fruit Website

Project Type: 2024 Final Exam Project

Platform: Website

My role: UI/UX and Web developer

Tools used: HTML, JavaScript, CSS and D3.js

Description

I designed a fruit-themed e-commerce website centered around a specific theme using public APIs. I worked as the sole UI/UX developer and web developer, handling both the design and technical parts of the website which included the planning, research, wireframes, and code.

The website allows users to purchase fruit-based diet plans and emphasizes credibility through customer testimonials and informative content that supports safe purchasing decisions. Additionally, the site features a D3.js data visualization that presents weight gain and weight loss trends, helping users identify which fruits are suitable for their body goals.

Competitive Analysis

Before starting the project, I did a competitive analysis to identify the design patterns and trends of existing food based subscription websites to better understand the user flow and packaging of these websites.

These websites included

  • Hello Fresh
  • Blue Apron
  • Gold Belly
  • Sun Basket

To better understand the target user, I began by researching common health conditions affecting people in South Africa. This research was conducted using academic sources found on Google Scholar, which highlighted that many South Africans face diet-related health challenges such as obesity, diabetes, hypertension, and vitamin deficiencies. These conditions are often linked to limited access to nutritious food, affordability issues, and a lack of clear, practical dietary guidance.

Through this research, I identified that fruit plays an important role in addressing several of these health concerns, as it is a natural source of essential vitamins, minerals, and fibre. However, despite its benefits, fruit consumption is often inconsistent due to factors such as cost, convenience, lack of education around nutritional value, and uncertainty about how to incorporate fruit into daily meals.

This insight informed the direction of the website by emphasizing accessibility, education, and simplicity. The platform aims to help users make informed, healthier food choices by clearly communicating the benefits of fruit, offering practical usage ideas, and presenting information in a way that is easy to understand and culturally relevant to a South African audience.

Before beginning the development phase, I created low-fidelity and mid-fidelity wireframes to better understand how users would navigate the website. This process helped me visualise content placement, refine functionality, and identify potential usability issues early in the design process.

Problem 1: API Down

At first I had a public API that had information regarding fruit nutrients and other essential details but after a week the API was temporarily unavailable. The API was essential in displaying my data visualizations.

Solution

Changing the theme of my website was not an option so I decided to take a more medical approach that displayed Covid-19 statistics to emphasize the need for users to invest in healthy living and show them the need to opt for healthier options such as fruits which improve their immune system which is essential.

Problem 2: Hard information to understand

Information about nutrition is often too scientific or overwhelming leading to users not knowing which fruits help with specific health needs (e.g. energy immunity).

Solution

I simplified the content using plain language, and highlighted the benefits of eating fruits into 10 points.

Reflection

Learning

Research-led design:Academic and contextual research helped me design based on real user needs, not assumptions.

Simplifying information:Breaking down complex nutritional info into clear, scannable content improved usability.

Wireframing early:Low and mid-fidelity wireframes revealed layout and navigation issues before high-fidelity design.

Design influence behaviourVisual hierarchy, colour, and content placement guided users toward healthier choices.

Challenges

Mobile optiization:Designing an interface that works well on small screens while keeping content clear was a challenge.

Content priortization:Deciding which fruits, health tips, and features to highlight first required careful planning.
